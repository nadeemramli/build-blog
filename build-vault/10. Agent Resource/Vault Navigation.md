# Vault Navigation

## Mental Model

The Build Vault is the practical workspace for building, launching, operating, and learning from hustle projects. The separate Career Path vault is for theory, certifications, and general learning. This vault is for applied knowledge and real-world execution.

## Root Folders

| Folder | Use it for | Agent posture |
| --- | --- | --- |
| `1. Proposition` | Value, ideation, PVP, evaluation, and deciding what is worth building. | Use when shaping or judging an idea. |
| `2. Distribution` | Growth, channels, content, marketing, and demand creation. | Use when finding customers or building GTM. |
| `3. Operation` | Engineering, product management, ecommerce systems, programming, and execution process. | Use when turning plans into systems or workflows. |
| `4. Economic` | Business models, analytics, finance, antifragility, and metrics thinking. | Use when modelling money, metrics, and viability. |
| `5. Idea Vault` | Project knowledge, PRDs, task lists, active/draft/archived project notes. | Use as the main project management layer. |
| `6. Ethics` | Principles discovered during building. | Use for judgment, boundaries, and operating philosophy. |
| `7. Canvas` | Workflow and canvas-style notes. | Use for visual/process exploration. |
| `8. Building Blocks` | Business management frameworks, agent operating files, automation philosophy, templates. | Use for reusable management systems. |
| `9. Real World` | Live implementation and operating records. | Use carefully; this is where real business state lives. |
| `10. Agent Resource` | Agent navigation, operating memory, and changelog. | Read first, update when the agent workflow changes. |
| `Template` | Reusable note templates. | Copy structure from here when creating recurring note types. |
| `Resource` | Supporting resources. | Read when referenced by another note. |

## Money and Budgeting Rule

For budgeting or money work, first identify whether the work is conceptual or operational:

- Conceptual modelling: start in `4. Economic`.
- Project-level economic planning: start in the project folder under `5. Idea Vault`.
- Live business records: start in `9. Real World`, then use that business's operating manual and data model.

For INDEXA, the key money area is `9. Real World/INDEXA/03 - Capital`. Read `00 - Business OS/Data Model & Conventions.md` and `00 - Business OS/Agent Operating Manual.md` before creating or changing money records. Never invent amounts, stock counts, revenue, COGS, or reconciliation status.

## Project State Rule

Do not assume every folder under `Active` is actively being pursued today. The current confirmed active pursuits are Canvasm/Metrimap and INDEXA. Other folders may be active, paused, archival residue, or unclassified until the user confirms them.
