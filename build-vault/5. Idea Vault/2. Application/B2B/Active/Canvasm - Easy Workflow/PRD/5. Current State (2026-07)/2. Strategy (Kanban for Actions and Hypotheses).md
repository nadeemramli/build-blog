Not mentioned anywhere in this vault's PRDs — this is a real, shipped feature area with no prior spec here.

## What it is

A dedicated page (separate from the canvas) that gives Action Nodes and Hypothesis Nodes — the `Work/Action` and `Ideas/Hypothesis` card categories from the original Card Category system — a project-management view instead of only living as cards on the canvas.

## What's shipped

- **Unified kanban board** for Action + Hypothesis nodes across a canvas (or workspace, tbc against actual scope) — `StrategyBoard.tsx`.
- **Monday.com-style table view** as an alternative to the board, with inline editing — `StrategyTable.tsx`. This was the largest single recent commit in the repo (~1,100 lines).
- **Due dates, assignees, comments** per card — `DueDateCell.tsx`, `AssigneeCell.tsx`, `CardCommentSheet.tsx`.
- **Value journey strip** — a visual strip tying strategy items back to the value chain — `ValueJourneyStrip.tsx`, `valueJourney.ts`.
- Inline-editable pills for status/tags — `EditablePill.tsx`.

## Why it matters for this vault

The vault's PRDs treat Action/Hypothesis as just another card category with no dedicated surface. In practice this grew into its own page with board + table views — closer to a lightweight project-management tool bolted onto the metric tree than a passive card type. Worth reflecting in any future homepage/IA doc (e.g. if `4. Product/1.a Homepage.md`'s navigation sketch is revisited, Strategy should get its own sidebar entry alongside Canvas/Dashboard/Assets/Source).

## Not yet known / worth checking with the team
