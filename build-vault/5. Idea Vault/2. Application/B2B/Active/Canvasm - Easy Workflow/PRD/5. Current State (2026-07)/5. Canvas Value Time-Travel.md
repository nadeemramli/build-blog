Not covered anywhere in this vault. The old PRDs' closest concept is the Relationship "Influence Drift" changelog (a chart of a relationship's weight over time) — this is the newer, canvas-wide equivalent for metric *values*, not just relationship strength.

## What it does

Lets you view the whole canvas as it looked at a past point in time, using the Tracked-Metric snapshot store described in the Object Model doc.

## How it works

- A **"Live"** control (History icon) appears in the canvas header — hidden entirely if the canvas has no time-series data to travel through.
- **View as of** a past period: every metric card shows its value as of that period (series truncated to that point). A card with no data yet at that point shows "No data as of <period>" rather than a zero.
- **Compare with** a second period: cards show the delta (absolute + %) between the two periods, green for up / red for down.
- **Reset to live** returns everything to current values.
- Switching canvases resets time-travel back to Live (it doesn't silently carry over).

## What it does NOT do (yet)

- Alert badges (see the Alerting doc) always reflect the live value — time-traveling doesn't change what's flagged as breaching.
- Dashboard KPI tiles still show latest only — time-travel is canvas-only for now, not yet wired into Dashboards.

## Current status (as of 2026-07-04)

Shipped, manually tested per `docs/tests/manual-test-plan.md` §12. Depends on the two-tier value storage in the Object Model doc — only Tracked Metrics (catalogued, source-bound) accumulate the snapshot history needed for this; purely exploratory cards have nothing to travel through.
