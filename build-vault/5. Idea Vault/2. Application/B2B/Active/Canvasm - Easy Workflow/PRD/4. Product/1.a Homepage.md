## **PRD: Figma-Inspired Workspace & Organizations**

- **Date**: August 11, 2025
    
- **Version**: 1.0
    
- **Status**: Draft
    

### **1. Introduction**

This document details the requirements for transforming the Canvasm homepage into a collaborative, Figma-style workspace. The current homepage serves as a "mission control" or gallery for managing canvas projects. While functional, it lacks the dynamic, team-oriented environment that modern creative and analytical tools provide.

To foster true collaboration and create a clear monetization path, we will introduce **Organizations** as a core concept. The homepage will be redesigned from a static list into an interactive workspace that surfaces teams, projects, and shared resources, making it the central nervous system for a team's data operations.

### **2. Goals & Objectives**

- **Primary Goal**: Evolve the homepage from a project launcher into a collaborative workspace that increases user engagement and drives team adoption.
    
- **Objectives**:
    
    - Introduce a paid **Organization** tier to establish a clear monetization strategy.
        
    - Create a fluid and intuitive project Browse and organization experience, similar to Figma's file browser.
        
    - Visually represent real-time collaboration directly on the homepage (e.g., showing who is active in a canvas).
        
    - Clarify the distinction between personal drafts and shared organizational projects.
        

---

### **3. Core Feature Specifications**

#### **3.1. The New Workspace View**

The grid/list of canvases will be replaced by a more interactive "meta-canvas" file browser.

- **Layout**: The main area will be a file browser. It will still support list and thumbnail views for canvases, but it will be nested within a project and team structure.
    
- **Projects**: Within an Organization or a user's personal space, users can create **Projects**. These are folders used to organize related canvases (e.g., "Q3 Growth Experiments," "Marketing Funnel Analysis"). This is an evolution of the simple "Labels/Tags" system.
    
- **Breadcrumb Navigation**: A breadcrumb trail (e.g., `SaaSCo Organization > Marketing Team > Q3 Growth Experiments`) will always be visible at the top, allowing users to navigate the hierarchy easily.
    

#### **3.2. Organizations: The Collaboration Hub (Paid Feature)**

Organizations are the top-level container for teams, projects, shared resources, and billing.

- **Creation & Tiers**: Any user can create an Organization, which will move them onto a paid plan.
    
    - **Free/Personal Tier**: A user's default space. Canvases here are considered "Drafts" by default. Collaboration is limited (e.g., invite up to 2 editors per canvas).
        
    - **Organization Tier (Paid)**: Unlocks the ability to create teams, have unlimited collaborators, set advanced permissions, and manage centralized billing.
        
- **Organization Homepage**: When inside an Organization, the homepage displays all the teams and projects belonging to it. It will have its own settings for members, billing, and shared assets.
    
- **Teams**: Within an Organization, admins can create **Teams** (e.g., "Product," "Growth," "Data Science"). Canvases and projects can be assigned to specific teams, controlling visibility and access rights.
    

#### **3.3. Enhanced Left Sidebar Navigation**

The homepage will feature a new, persistent sidebar that provides the main navigation for the workspace. This is distinct from the in-canvas sidebar which appears _after_ opening a canvas.

- **User Profile/Settings**: At the top, showing the current user.
    
- **Search**: A global search bar to find canvases, projects, or teams.
    
- **Recents**: A quick-access list of the last few canvases the user viewed.
    
- **Personal Space**:
    
    - **Drafts**: A dedicated section for a user's personal, non-organizational canvases. This is the default location for new canvases created in the free tier.
        
- **Organization(s) Section**:
    
    - Lists all Organizations the user is a member of.
        
    - Clicking on an Organization name expands it to show the **Teams** within it.
        
    - Clicking on a Team name filters the main workspace view to show only the projects and canvases belonging to that team.
        

#### **3.4. Dynamic Canvas Cards**

The representation of each canvas project on the homepage will be enhanced to feel more alive and informative, evolving from the initial concept.

- **Live Avatars**: If a canvas is being actively viewed or edited, the avatars of the current users will be shown on the card.
    
- **Sharing Indicator**: A clear icon indicating the sharing status of the canvas (e.g., "Shared with Organization," "Shared with Marketing Team," "Private").
    
- **Last Edited**: Shows "Edited 5 minutes ago by [User Avatar]" instead of just a static "Last Modified Date".
    
- **Context Menu**: Right-clicking a canvas card will bring up a context menu with options like `Share`, `Rename`, `Move to project...`, `Duplicate`, and `Delete`.

### **4. User Flow Examples**

- **Creating a Collaborative Project**:
    
    1. User clicks "Create New Organization" from the sidebar.
        
    2. They are prompted with the pricing plan and enter billing details.
        
    3. The Organization ("SaaSCo") appears in their sidebar.
        
    4. Inside "SaaSCo," they create a new Project called "Q3 Growth Model."
        
    5. They invite team members to the Organization.
        
    6. They create a new canvas within the project, and all team members instantly have access to view and edit it in real-time.
        
- **From Personal to Team**:
    
    1. A user creates a new canvas in their personal "Drafts."
        
    2. They build out an initial model.
        
    3. Once ready for feedback, they right-click the canvas card and select "Move to project..."
        
    4. They choose the "Q3 Growth Model" project within their paid Organization.
        
    5. The canvas is now moved from their private drafts to the shared team space, inheriting the project's sharing permissions.
        

### **5. Out of Scope for this Version**

- **Figma-style Community Page**: A public repository for sharing canvas templates is not included in this phase.
    
- **Complex Permission Roles**: Initial roles will be simple (e.g., Admin, Editor, Viewer). Granular, per-node permissions will be considered later.
    
- **In-depth Billing Management**: The initial billing system will be basic. Features like seat-based pricing adjustments and detailed invoice history will follow.


Based on the PRD for the new Figma-inspired workspace, there are **11 distinct routes** a user can take from the homepage.

These routes are accessible through the enhanced left sidebar, the main workspace view, and the dynamic canvas cards.

---

### **Left Sidebar Routes**

1. **Global Search**: Initiates a search across all canvases, projects, and teams.
    
2. **Recents**: Navigates to a filtered view showing only recently accessed canvases.
    
3. **Drafts**: Takes the user to their personal space, showing all private canvases not assigned to an organization.
    
4. **Create New Organization**: Starts the workflow for creating a new paid organization, likely routing to a setup or billing page.
    
5. **Organization Homepage**: Navigates to the main page for a specific organization.
    
6. **Team Homepage**: Filters the workspace to show only the projects and canvases belonging to a specific team within an organization.
    

### **Main Workspace & Canvas Card Routes**

7. **Open Canvas**: The primary action of clicking on a canvas card to open the editor view.
    
8. **Open Project**: Navigates into a project folder to view the canvases contained within it.
    
9. **User Profile/Settings**: Accesses the user's account settings, profile information, and potentially personal billing.
    
10. **Share Canvas**: Opens a sharing modal or page to manage collaborators and permissions for a specific canvas.
    
11. **Move to Project**: Initiates a workflow to move a canvas from its current location (e.g., Drafts) into a specific project folder within an organization.
