# Shared Infra (Hetzner VPS) — Design / Ops

**One warm box for the serverless-hostile half of the roadmap.** The long-lived / heavy / stateful / scheduled workloads need an always-on process — which is exactly what Vercel serverless and Supabase Edge Functions are not. A small Hetzner VPS + **Docker Compose** hosts them together: cheap, flexible, no lock-in, with the ops tax owned by us. Managed DB/auth stay on Supabase.

## Co-tenants
Linear project: *Shared infra & workers (Hetzner VPS)* (under the Development Operating System initiative).

## NOT on the VPS
- Primary **Supabase Postgres** — keep managed + backed up.
- **Static sites** (marketing/docs) → Cloudflare/Vercel Pages.
- **Analytics** → **hosted PostHog** (CVS-115). **Not self-hosted** (decided).

## Guardrails
- **RLS-scoped path** for anything touching metric data (no service-role bypass) — same rule as the MCP (CVS-118). Compute worker, drift engine, and export renderer all respect node visibility.
- **Containerize** each service (isolation); back up; it's a single point of failure + security surface → keep patched, size up rather than cram.
- MCP transport: **Streamable HTTP** (SSE is deprecated).

## Provision order
CVS-215 (provision) → CVS-216 (queue) → workers (217 / 218 / 220) + export (219) + CRDT (221) + Loop-B (222). MCP (CVS-100), connected-accounts (CVS-90), and ingest (CVS-102) deploy onto it.
