# 2026-07-09 - Twinagent Project Start

## Prompt

The user wants to create something like Agent Notch, almost like a Superwhisper UI, where a user can hover to manage agent work.

The first desired use case is logging Codex and Claude into it to see how agents are working. It should also integrate with VPS agents and other active agents, so running work, monitoring, scheduling, and needs-user moments are visible in one glance.

## Reference Direction

The supplied screenshots show:

- A compact notch-style panel.
- Agent count and usage summary.
- Needs-you alerts.
- Context or plan usage gauges.
- Agent/project cards.
- Working and blocked states.
- Click-to-jump behavior.
- Live subagents.
- Inline approvals.
- 100 percent local positioning.

The external page was referenced as [Agent Notch](https://www.agentnotch.app/). The environment did not return useful page contents during setup, so the first capture is based on the user's screenshots and description.

## Initial Decision

Create `Twinagent` under the active B2C idea vault as an early concept with:

- Project hub.
- PRD.
- Task list.
- Reference note.

## Early Product Thesis

As agent work spreads across Codex, Claude, VPS workers, scheduled jobs, and local automations, the user needs a local command surface that answers:

- What is running?
- What needs me?
- What is close to a limit?
- What is scheduled next?
- Where do I jump back in?
