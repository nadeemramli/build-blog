# The Learning Loop (Personal Ritual)

The system-level learning loop — what to capture, where lessons route, the weekly review — lives in [[3.c Obsidian Learning Loop]]. This note is only my personal ritual and scoreboard: the part that measures whether *I* am improving as a builder, not whether the system is capturing knowledge.

The gap between someone who has used agents for a year and someone who has repeated one month twelve times is whether sessions **compound**. The test: does the same mistake happen twice?

## The 2-minute session close

After any substantial agent session, before closing the terminal:

> "Before we finish: what did we learn this session that the next session won't know? Update CLAUDE.md/AGENTS.md with repo facts, and give me anything vault-worthy."

The agent is better at summarizing the session than I am at remembering it next week. This costs two minutes and is the entire mechanism.

## Monthly self-audit (15 minutes)

Score each month, appended below:

1. **Repos:** any active directory failing the [[6. Project Bootstrap Protocol]]? (No git, no context files, no E2E.)
2. **Delegation level:** what % of agent tasks shipped without mid-course correction? (Rising % = moving Level 2 → 3 on the ladder in [[1. Claude Code Playbook]].)
3. **Verification:** did agents run the verify scripts themselves, or did I?
4. **Cross-review:** did risky changes get the second-model pass from [[2. Codex and Multi-Agent Strategy]]?
5. **Evaporation check:** name one lesson from this month. If I can't find where it's written, the loop leaked.
6. **Environment:** PATH still clean? Secrets still only in `~/.config/secrets/env`? Disk sane?
