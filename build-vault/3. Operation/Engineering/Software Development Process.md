Fundamentally what we do as a philosophy workflow is this:

1. We gather requirement and understand what needs to be done, and how we can work on it
2. We then code it, ensuring it's consistent, using the codebase language, using design principles, etc.
3. We then test to ensure the code is robust. 

That's all
